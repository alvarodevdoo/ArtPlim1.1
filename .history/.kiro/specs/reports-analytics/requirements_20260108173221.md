# Fase 4: Relatórios e Analytics - Requisitos

## Visão Geral

A Fase 4 implementa um sistema completo de relatórios e analytics para fornecer insights valiosos sobre performance, custos, margens e operações do sistema ERP. Esta fase transforma dados operacionais em informações estratégicas para tomada de decisão.

## Objetivos

1. **Dashboard Executivo**: Visão geral de KPIs e métricas importantes
2. **Relatórios de Custos e Margens**: Análise detalhada de rentabilidade
3. **Analytics de Materiais**: Otimização de uso e redução de perdas
4. **Previsão de Demanda**: Planejamento baseado em dados históricos
5. **Performance Tracking**: Monitoramento de eficiência operacional

## Requisitos Funcionais

### RF01 - Dashboard Executivo
- Exibir KPIs principais em tempo real
- Gráficos de vendas, produção e financeiro
- Comparações período a período
- Alertas para métricas fora do padrão
- Filtros por período, produto, cliente
- Export de dados para Excel/PDF

### RF02 - Relatórios de Custos e Margens
- Análise de rentabilidade por produto
- Comparação custo estimado vs real
- Margem de contribuição por pedido
- Análise de precificação
- Relatório de produtos mais/menos rentáveis
- Tendências de custos ao longo do tempo

### RF03 - Analytics de Materiais
- Análise de perdas por material
- Tracking de perdas reais vs estimadas
- Sugestões de otimização
- Relatório de consumo por produto
- Análise de eficiência de uso
- Alertas de desperdício excessivo

### RF04 - Previsão de Demanda
- Análise de padrões de consumo
- Previsão baseada em histórico
- Alertas de reposição de estoque
- Sazonalidade de produtos
- Tendências de crescimento/declínio
- Recomendações de compra

### RF05 - Performance Tracking
- Tempo médio de produção
- Taxa de aprovação de alterações
- Eficiência por operador
- Análise de gargalos
- Métricas de qualidade
- Indicadores de satisfação

## Estrutura de Dados

### Tabela: ReportConfigurations
```sql
CREATE TABLE ReportConfigurations (
  id UUID PRIMARY KEY,
  organizationId UUID NOT NULL,
  userId UUID NOT NULL,
  name VARCHAR(255) NOT NULL,
  type ENUM('DASHBOARD', 'COST_ANALYSIS', 'MATERIAL_ANALYTICS', 'DEMAND_FORECAST'),
  filters JSONB NOT NULL,
  schedule JSONB, -- Para relatórios automáticos
  isPublic BOOLEAN DEFAULT FALSE,
  createdAt TIMESTAMP DEFAULT NOW(),
  updatedAt TIMESTAMP DEFAULT NOW()
);
```

### Tabela: ReportCache
```sql
CREATE TABLE ReportCache (
  id UUID PRIMARY KEY,
  organizationId UUID NOT NULL,
  cacheKey VARCHAR(255) NOT NULL,
  data JSONB NOT NULL,
  expiresAt TIMESTAMP NOT NULL,
  createdAt TIMESTAMP DEFAULT NOW()
);
```

### Views Materializadas para Performance
```sql
-- View para métricas de vendas
CREATE MATERIALIZED VIEW SalesMetrics AS
SELECT 
  organizationId,
  DATE_TRUNC('day', createdAt) as date,
  COUNT(*) as order_count,
  SUM(total) as total_revenue,
  AVG(total) as avg_order_value
FROM Order 
WHERE status != 'CANCELLED'
GROUP BY organizationId, DATE_TRUNC('day', createdAt);

-- View para análise de custos
CREATE MATERIALIZED VIEW CostAnalysis AS
SELECT 
  o.organizationId,
  oi.productId,
  p.name as product_name,
  DATE_TRUNC('month', o.createdAt) as month,
  SUM(oi.costPrice * oi.quantity) as total_cost,
  SUM(oi.unitPrice * oi.quantity) as total_revenue,
  SUM((oi.unitPrice - oi.costPrice) * oi.quantity) as total_margin
FROM Order o
JOIN OrderItem oi ON o.id = oi.orderId
JOIN Product p ON oi.productId = p.id
WHERE o.status = 'DELIVERED'
GROUP BY o.organizationId, oi.productId, p.name, DATE_TRUNC('month', o.createdAt);
```

## Métricas e KPIs

### Dashboard Executivo
- **Receita Total**: Soma de pedidos entregues
- **Margem Bruta**: (Receita - Custos) / Receita
- **Ticket Médio**: Receita / Número de pedidos
- **Taxa de Conversão**: Pedidos aprovados / Total de orçamentos
- **Pedidos em Andamento**: Por status
- **Top 5 Produtos**: Por receita e margem
- **Top 5 Clientes**: Por volume e valor

### Análise de Custos
- **Custo por Produto**: Detalhamento de materiais e mão de obra
- **Variação de Custos**: Comparação período a período
- **Margem por Categoria**: Agrupamento por tipo de produto
- **Eficiência de Precificação**: Preço praticado vs sugerido
- **Produtos Deficitários**: Margem negativa ou muito baixa

### Analytics de Materiais
- **Taxa de Perda**: Perda real / Consumo total
- **Eficiência de Uso**: Aproveitamento por material
- **Custo de Desperdício**: Valor monetário das perdas
- **Materiais Críticos**: Maior impacto em custos
- **Tendência de Perdas**: Evolução ao longo do tempo

### Performance Operacional
- **Tempo Médio de Produção**: Por produto e complexidade
- **Taxa de Alterações**: Alterações / Total de pedidos
- **Tempo de Aprovação**: Média de aprovação de alterações
- **Produtividade**: Pedidos processados / Tempo
- **Qualidade**: Taxa de retrabalho e reclamações

## Alertas e Notificações

### Tipos de Alertas
- **Margem Baixa**: Produtos com margem < 20%
- **Perda Excessiva**: Material com perda > meta
- **Demanda Anômala**: Variação > 50% da média
- **Estoque Baixo**: Previsão de ruptura
- **Performance Degradada**: KPIs fora do padrão

### Canais de Notificação
- **Dashboard**: Alertas visuais na interface
- **Email**: Relatórios automáticos
- **WebSocket**: Notificações em tempo real
- **Mobile**: Push notifications (futuro)
