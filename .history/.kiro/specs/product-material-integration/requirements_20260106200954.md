# Requirements Document

## Introduction

Este documento especifica os requisitos para implementar um sistema completo de vinculação entre produtos e materiais, permitindo que produtos sejam configurados com seus componentes materiais e que o sistema calcule automaticamente custos, consumo e disponibilidade de estoque.

## Glossary

- **Product**: Produto do catálogo que pode ser vendido aos clientes
- **Material**: Matéria-prima utilizada na produção (lona, vinil, ACM, etc.)
- **ProductComponent**: Vinculação entre um produto e um material específico
- **ConsumptionMethod**: Método de cálculo de consumo do material (BOUNDING_BOX, LINEAR_NEST, FIXED_AMOUNT)
- **MaterialCalculator**: Componente que calcula consumo e custos de materiais
- **PricingEngine**: Sistema que calcula preços baseado em custos de materiais e operações
- **InventoryItem**: Item físico de material em estoque

## Requirements

### Requirement 1: Gerenciamento de Componentes de Produto

**User Story:** Como um administrador, eu quero vincular materiais aos produtos, para que o sistema possa calcular automaticamente os custos de produção.

#### Acceptance Criteria

1. WHEN um usuário acessa a página de edição de produto, THE System SHALL exibir uma seção para gerenciar componentes de material
2. WHEN um usuário adiciona um material ao produto, THE System SHALL permitir selecionar o método de consumo e percentual de perda
3. WHEN um usuário remove um componente de material, THE System SHALL confirmar a ação e atualizar o produto
4. THE System SHALL validar que pelo menos um material seja obrigatório para produtos com modo DYNAMIC_ENGINEER
5. WHEN um material é vinculado a um produto, THE System SHALL salvar a configuração na tabela ProductComponent

### Requirement 2: Interface de Seleção de Materiais

**User Story:** Como um usuário, eu quero selecionar materiais de uma lista organizada, para que eu possa configurar produtos facilmente.

#### Acceptance Criteria

1. WHEN um usuário clica em "Adicionar Material", THE System SHALL exibir um modal com lista de materiais disponíveis
2. THE System SHALL filtrar materiais por nome, formato e disponibilidade
3. WHEN um usuário seleciona um material, THE System SHALL exibir campos para método de consumo e percentual de perda
4. THE System SHALL validar que o método de consumo seja compatível com o formato do material
5. WHEN um usuário confirma a adição, THE System SHALL criar o ProductComponent e atualizar a interface

### Requirement 3: Cálculo Real de Materiais

**User Story:** Como um usuário, eu quero que o MaterialCalculator use dados reais do sistema, para que os cálculos sejam precisos e atualizados.

#### Acceptance Criteria

1. WHEN o MaterialCalculator é aberto para um item de pedido, THE System SHALL buscar os materiais vinculados ao produto
2. THE System SHALL calcular o consumo baseado no método configurado para cada material
3. WHEN não há materiais vinculados, THE System SHALL exibir uma mensagem informativa
4. THE System SHALL verificar o estoque real disponível para cada material
5. WHEN o estoque é insuficiente, THE System SHALL exibir alertas específicos

### Requirement 4: Visualização de Materiais em Pedidos

**User Story:** Como um usuário, eu quero ver quais materiais são necessários para cada item do pedido, para que eu possa planejar a produção.

#### Acceptance Criteria

1. WHEN um pedido é visualizado, THE System SHALL exibir os materiais necessários para cada item
2. THE System SHALL mostrar a quantidade calculada e o custo de cada material
3. WHEN há problemas de estoque, THE System SHALL destacar os materiais em falta
4. THE System SHALL permitir reservar materiais diretamente da visualização do pedido
5. WHEN o produto não tem materiais configurados, THE System SHALL exibir um aviso

### Requirement 5: APIs para Gerenciamento de Componentes

**User Story:** Como desenvolvedor, eu quero APIs para gerenciar componentes de produto, para que o frontend possa interagir com os dados.

#### Acceptance Criteria

1. THE System SHALL fornecer endpoint POST /api/catalog/products/:id/components para adicionar componentes
2. THE System SHALL fornecer endpoint DELETE /api/catalog/products/:id/components/:componentId para remover componentes
3. THE System SHALL fornecer endpoint GET /api/catalog/products/:id/components para listar componentes
4. THE System SHALL validar que o material existe antes de criar o componente
5. THE System SHALL retornar erros apropriados quando operações falham

### Requirement 6: Integração com PricingEngine

**User Story:** Como o sistema, eu quero que o PricingEngine use os materiais reais vinculados aos produtos, para que os cálculos de preço sejam precisos.

#### Acceptance Criteria

1. WHEN um produto tem modo DYNAMIC_ENGINEER, THE PricingEngine SHALL usar apenas os materiais vinculados
2. THE PricingEngine SHALL calcular o consumo usando o método configurado para cada componente
3. WHEN um produto não tem materiais configurados, THE PricingEngine SHALL retornar erro específico
4. THE PricingEngine SHALL aplicar o percentual de perda configurado para cada material
5. THE PricingEngine SHALL somar todos os custos de materiais e operações

### Requirement 7: Validação de Estoque

**User Story:** Como um usuário, eu quero ser alertado quando não há material suficiente, para que eu possa tomar ações preventivas.

#### Acceptance Criteria

1. WHEN um pedido é criado, THE System SHALL verificar disponibilidade de materiais
2. THE System SHALL calcular o consumo total considerando todos os itens do pedido
3. WHEN o estoque é insuficiente, THE System SHALL exibir alerta detalhado
4. THE System SHALL sugerir materiais alternativos quando disponível
5. THE System SHALL permitir criar o pedido mesmo com estoque insuficiente, mas com aviso

### Requirement 8: Reserva de Materiais

**User Story:** Como um usuário, eu quero reservar materiais para pedidos aprovados, para que eles não sejam utilizados em outros pedidos.

#### Acceptance Criteria

1. WHEN um pedido é aprovado, THE System SHALL permitir reservar materiais automaticamente
2. THE System SHALL criar movimentações de estoque do tipo RESERVED
3. WHEN materiais são reservados, THE System SHALL atualizar a disponibilidade
4. THE System SHALL permitir cancelar reservas quando pedidos são cancelados
5. THE System SHALL exibir materiais reservados na visualização do estoque

### Requirement 9: Relatório de Consumo de Materiais

**User Story:** Como um gerente, eu quero relatórios de consumo de materiais por produto, para que eu possa otimizar custos e estoque.

#### Acceptance Criteria

1. THE System SHALL gerar relatório de consumo de materiais por período
2. THE System SHALL mostrar quais produtos consomem mais de cada material
3. THE System SHALL calcular custos totais de materiais por produto
4. THE System SHALL identificar materiais com maior desperdício
5. THE System SHALL permitir exportar relatórios em formato Excel

### Requirement 10: Configuração de Métodos de Consumo

**User Story:** Como um administrador, eu quero configurar como cada material é consumido, para que os cálculos sejam precisos para diferentes tipos de produção.

#### Acceptance Criteria

1. THE System SHALL permitir configurar método BOUNDING_BOX para materiais em chapa
2. THE System SHALL permitir configurar método LINEAR_NEST para materiais em rolo
3. THE System SHALL permitir configurar método FIXED_AMOUNT para materiais unitários
4. THE System SHALL validar que o método é compatível com o formato do material
5. THE System SHALL permitir configurar percentual de perda específico por componente
